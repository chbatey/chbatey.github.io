But where do Cassandra and Dynamo differ?

Data model

Well this is the obvious one. The Dynamo paper describes a key value store where as Cassandra uses  Columnar (wide row) data model. This is less apparent in the later versions of Cassandra where CQL introduces what look like relational tables/rows, however the wide row feature is still supported by having many CQL rows stored together in the same partition.

Hinted handoffs 

Both Dynamo and Cassandra aim to be tolerant of nodes being down in the cluster with the notion of self repairing when the nodes comes back up. The mechanism for achieving this is referred to as hinted handoffs for both systems but they differ slightly.
