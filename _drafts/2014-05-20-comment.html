Drop wizard recommends using maven so lets create a simple project:

mvn archetype:generate -DarchetypeGroupId=org.apache.maven.archetypes -DgroupId=com.batey.examples -DartifactId=dropwizard-restapp

Then we need to add the dropwizard dependency:
<dependencies>
    <dependency>
        <groupId>io.dropwizard</groupId>
        <artifactId>dropwizard-core</artifactId>
        <version>${dropwizard.version}</version>
    </dependency>
</dependencies>

Building a shaded fat jar:
Drop wizard recommend using the maven shade and jar plugin to create a fat jar. Once you have added the below to your pom you can run mvn package and it will build an executeable jar.

Before you can run the jar you need to create a config file that matches your configuration class. All we need for our example is the following:

numberToStore: 500

Now it is time to run it, if you don't pass a config file in you get a nice error message:

java -jar ./target/dropwizard-restapp-1.0-SNAPSHOT.jar server
default configuration has an error:
  * numberToStore must be greater than or equal to 1 (was 0)

Now adding the config file the server starts up:

java -jar ./target/dropwizard-restapp-1.0-SNAPSHOT.jar server config.yml
